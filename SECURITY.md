# Security Policy

Ratatosk is still in alpha. The project may change quickly, and security hardening is ongoing. If you find a vulnerability, please report it responsibly so it can be investigated and fixed before details are disclosed publicly.

## Supported Versions

At this stage, security fixes should be expected only for:

- The latest commit on `main`
- The most recent release, once releases exist

Older commits, forks, and modified deployments may not receive fixes.

## Reporting a Vulnerability

Please do not report undisclosed vulnerabilities in public GitHub issues, discussions, or pull requests.

Instead, use one of the following private channels:

- GitHub private vulnerability reporting, if it is enabled for this repository
- A private maintainer contact channel, if one is published by the project

If neither option is available, limit public reports to non-sensitive information and avoid publishing exploit details, proof-of-concept code, secrets, or reproduction steps that would make active abuse easier.

## What to Include

When reporting a vulnerability, include as much of the following as possible:

- A clear description of the issue and affected component
- The impact and realistic attack scenario
- Steps to reproduce
- Any proof of concept, logs, or packet traces that help confirm the issue
- The commit SHA, branch, or build you tested
- Any relevant environment details, such as OS, architecture, network setup, or proxy configuration

## Disclosure Process

Maintainers will review reports, validate the issue, assess impact, and work on a fix as time permits.

Please allow time for investigation and remediation before public disclosure. Once a fix is available, maintainers may disclose the issue publicly with appropriate technical detail and credit, unless you request to remain anonymous.

## Scope

This policy covers security issues in the Ratatosk codebase and its default behavior. It does not automatically cover:

- Third-party services or infrastructure not maintained by this project
- Vulnerabilities caused only by downstream modifications
- General support questions, configuration help, or feature requests

If you are unsure whether something is a security issue, report it privately anyway.
